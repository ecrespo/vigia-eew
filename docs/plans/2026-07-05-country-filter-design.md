# Country notification filter (RF-37) — design

Status: approved 2026-07-05, implemented.

## Problem

The user wants to avoid being notified about earthquakes in **other countries**: only
alert on quakes in the user's own country (with the existing radius/magnitude still
applying). Near borders and coasts the plain radius filter isn't enough — a quake 80 km
away can be in a neighbouring country (Colombia, Trinidad, Caribbean islands).

## Decisions (from brainstorming)

1. **Semantics — block-list, not allow-list.** Drop an event only if it falls
   **positively inside another country**. Ocean / offshore / unknown-country points are
   **kept** (subject to radius + magnitude). This is essential because Venezuela's most
   dangerous quakes are offshore ("NEAR COAST OF VENEZUELA"), outside any land polygon; a
   strict "must be inside my country" rule would drop exactly those.
2. **How to determine a quake's country — bundled offline boundaries + pure-Python
   point-in-polygon.** Ship a reduced Natural Earth 1:110m dataset
   (`assets/countries.geojson`, public domain, ~186 KiB, `ISO_A2` + geometry). No
   geospatial dependency (ray casting + bbox pre-check), matching the project's
   zero-extra-deps default and the "bundle a generated asset" pattern (wav/icon).
3. **User's country — derived from the resolved reference point** via the same
   reverse-geocoder (`country_of(reference.lat, reference.lon)`), so it works whether the
   reference is manual or IP-detected, with no extra network call. Config override
   `[filter] country = "VE"`; `"auto"` (default) derives it.
4. **Opt-in, off by default.** New `[filter] country_filter = false`. This is a safety
   tool: enabling a filter that could misclassify a country risks silencing a real alert,
   so the user turns it on explicitly. Fail-safe throughout: if the country can't be
   determined, the filter stays inert (never suppresses).

## Components

- `geocode.py` — `country_of(lat, lon) -> str | None` (ISO-A2 or None). `load_index()`
  builds a `[(iso, bbox, polygon)]` index from a GeoJSON FeatureCollection; pure-Python
  ray casting with hole support; bundled dataset loaded once, lazily, cached.
- `assets/countries.geojson` — generated by `packaging/build_countries_geojson.py`
  (downloads Natural Earth, strips to `{iso, geometry}`, rounds coords to 3 dp).
- `config.py` — `Filter.country_filter: bool = False`, `Filter.country: str = "auto"`.
- `pipeline/filter.py` — `GeoFilter(cfg, *, user_country=None, country_of=None)`. Rejects
  an event only when `country_of(ev.lat, ev.lon)` is a country other than `user_country`.
  Inert (backward compatible) when the injected params are absent.
- `app.py` — `_resolve_user_country()` (config override or reverse-geocode the reference)
  and `_build_geo_filter()` (wires the country filter when enabled; fail-safe to inert).
  Used by both `execute()` and `run_tui()`; not by `--simulate`.

## Testing

Pure-Python point-in-polygon tested with synthetic polygons (square country + hole,
MultiPolygon) plus smokes against the real asset (Caracas→VE, Bogotá→CO, mid-Atlantic→
None). Filter tested with an injected `country_of`: rejects other-country, keeps
same-country / offshore(None) / disabled / no-user-country / never overrides radius.
App wiring tested for config-override vs auto-from-reference and the inert fail-safe.
End-to-end pipeline run confirms: Caracas kept, Bogotá/Trinidad/Lima dropped, offshore VE
kept.

## Trade-offs / future

- 1:110m resolution is coarse near borders (±tens of km); upgrade path to 1:50m if border
  precision matters (bigger asset). Acceptable for a best-effort filter, off by default.
- No new pip dependency; the boundary data is a bundled asset (RNF-06 unaffected).
